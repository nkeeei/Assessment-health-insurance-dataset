4. Построение модели предсказания и оценка модели.¶

Предсказание модели я умею строить только по 2 признакам(x,y). Но, по предварительному анализу, можно сделать вывод, оценивать по какому-то особенному признакому расход не будет являться правильным и оптимальным.¶

Я решил, что для более правильного построения модели, выберу два признака. Один из признаков, конечно же, расход. Его мы и будем предсказывать. А второй признак создадим абсолютно новый, который будет опираться на данные нам параметры. То есть для всех имеющих признаков нужно определить вес. Второй признак нашей модели будет сумма произведений весов на показатели признаков. Выделим наиболее ценные признаки, от которых зависит расход: это конечно же 'smoker'. Количество детей так же имеет наглядные различия. Возраст можно отпустить, т.к. какую-либо зависимость от расходов мы не наблюдали. 5. Минимизация квадратичной ошибки.¶

В простейшей постановке задача прогноза значения вещественного признака по прочим признакам решается минимизацией квадратичной функции ошибки. Напишем функцию, которая по двум параметрам w0 и w1 вычисляет квадратичную ошибку приближения зависимости alleged_charges y от charges x прямой линией y=w0+w1∗x.¶

In [69]:
def get_y(w0, w1, x):
    return w0 + w1*x

def get_error(w0, w1):
    return sum(map(lambda x, y: (y - get_y(w0, w1, x)) ** 2, data.charges, data.alleged_charges))

Итак, мы решаем задачу: как через облако точек, соответсвующих наблюдениям в нашем наборе данных, в пространстве признаков "charges" и "alleged_charges" провести прямую линию так, чтобы минимизировать функционал. Для начала давайте отобразим хоть какие-то прямые и убедимся, что они плохо передают зависимость alleged_charges от charges. Проведите на последнем графике две прямые, соответствующие значениям параметров ( w0,w1)=(50,0.4) и ( w0,w1)=(9000,0.2).¶

С помощью метода оптимизации найдем "оптимальный" наклон прямой, приближающей зависимость прогнозированного расхода от действительного расхода, при фиксированном коэффициенте w0=9000 .¶

Используйте метод оптимизации L-BFGS-B (аргумент method метода minimize). Проведем на графике 1 прямую, соответствующую найденным оптимальным значениям параметров w0 и w1 .¶
